This section could be 1-10 pages.
A Methodology section makes the test more reproducible in the future. When the target organization in the future want redo the test, it can rely on this report to replicate it, performing a gap analysis on changes in the interim, or purposely tweak the process next time.

• Provide a description of the step-by-step process used, and briefly list the findings discovered at each phase.
• Include a table to briefly summarize the tools used in the penetration test.
• Include a table with one row per target machine, listing the IP address, machine name(s), associated business unit (if it can be determined), and its method of discovery
• If there aren't many vulnerable findings, thid is a good news but could question the quality and completeness of the test.
