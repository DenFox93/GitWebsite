2. Identification


How do you detect an incident?
With sensor platforms like firewalls, intrusion detection systems and system logs

Don't be afraid to declare an incident
Even if there is no attack you still help the organization. if it is been a false alarm, use these as training opportunities.

Assign Handlers
Assign a person to be the primary incident handler.
Ideally, assign a helper to him. This because if you have the resources, it's best to deploy two people to handle each incident to gather evidence

Communication Channels
◇ If the computers may have been compromised avoid using them for incident handling discussions (email or chat)
◇ Rely on out-of-band communications
▪ Use telephones and faxes
▪ Be careful with VoIP, which can be sniffed and played back using a variety of tools (Wireshark, Cain, and VOMIT) unless you have deployed VoIP with solid encryption.
◇ Make sure the team can send encrypted email, such as GnuPG, PGP, S/MIME, and so on
