Unattended Installs & Configuration files


In enterprise organizations are often used Unattended installs used to for automate the installation of Windows OSs.
If Windows administrators fail to properly clean up after this process, an XML file called "Unattend.xml" is left on the local system.

This file contains all the configuration settings that were set during the installation process, some of which can involve the configuration(like passwords in base 64!) of local accounts including Administrator accounts
