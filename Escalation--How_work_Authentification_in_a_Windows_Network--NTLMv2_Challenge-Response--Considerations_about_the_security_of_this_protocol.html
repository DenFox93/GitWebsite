Considerations about the security of this protocol


Considerations about the security of this protocol:



• Due to the Timestamp and Client Challenge, the Response of the client is different every time that is generated
• Impossible to create rainbow tables to gather the NT hash or the password from the NTLMv2 response.
• Dictionary does not make sense as the key (HMAC-MD5) is a hash.
• The only possible attack is by brute-forcing the HMAC-MD5 key
• The NTLMv2 hash is bound to a particular server and particular username so it’s not reusable.

After doing these considerations we can say that cracking NTLMv2 hashes is considered infeasible (if the password is strong enough).
Anyway we can still use the hash to mount different attacks
