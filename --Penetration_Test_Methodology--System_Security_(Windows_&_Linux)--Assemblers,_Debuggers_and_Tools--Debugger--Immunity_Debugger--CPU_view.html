CPU view


Interface of the CPU view

(Alt+c)
If we want to edit a value we have to double click on it

1. TOP LEFT PANEL(main window and most important panel): disassembled code of the program

first column: address location that contains the instructions
second column: opcode or machine code(hexadecimal value) of the instruction
third column: assembly code of each instruction
fourth column: created by the debugger and gives further information about the instructions(debugger comment)
with this line that comprehend multiple lines Immunity debugger define the start an the end of a function
2. TOP RIGHT PANEL: registers and their current values

first column: names of registers
second column: registers content
third column: if a register points to an ASCII string, the value of the string

3. BOTTOM LEFT PANEL: Memory dump of the entire program

first column: memory locations(addresses)
second column: content of the address in hexadecimal (is possible to show data in different formats: right click → Hex → ...)
third column: content of the address in ASCII

4. BOTTOM RIGHT PANEL: Stack view

We can note that the value at the top of the stack is the same value stored in the ESP register
first column: memory addresses location of the stack
second column: hexadecimal value(DWORD) contained by a memory address
third column: (ASCII or UNICODE or ...) representation
fourth column: more informations about the value contained(debugger comments)

☐ double click on an address we can see the relative offset of each address
