CPU, Assembly and ISA



Machine language


CPU(Central Processing Unit) is the device in charge of executing the machine code of a program.
machine code, or machine language, is a set of instructions in hexadecimal (HEX) format that the CPU processes
Each instruction is a primitive command that executes a specific operation

ISA: Instruction Set Architecture


Each type of CPU has its own instruction set architecture (ISA). ISA is what a programmer can see: memory, registers, instructions...
• x86 → This Instruction Set Architecture(ISA) is originated from the Intel 8086 and identifies 32-bit processors; 32 bit is the length of the registers in the CPU.
If the register(it is an address) is 32 bits, then there are 2^32 possible registers, and that means 2^32 (4.294.967.296) possible values.
Because one value in the register can reference an individual byte in memory, a 32-bit system can address a maximum of 4 gigabytes (4.294.967.296 bytes) of RAM
• x64 → This Instruction Set Architecture(ISA) also known as x86_64 or AMD64 identifies 64-bit processors; 64 bit is the length of the registers in the CPU.
If the register(it is an address) is 64 bits, then there are 2^64 possible registers, and that means 2^64 (18.446.744.073.709.551.616) possible values.
Because one value in the register can reference an individual byte in memory, a 64-bit system can address a maximum of 17.179.869.184 gigabytes (18.446.744.073.709.551.616 bytes) of RAM
