Weakness of the protocol


To exploit it we can go to the chapter Exploitation → Metasploit → Windows Authentication Attack → LM/NTLMv1 Attack

Weakness:


No Diffusion → each part of the Response is not linked to the previous one. This allows attacks on the three blocks individually

The third Response(RESPpart3) is much weaker than the others, since it has 5 null bytes for padding


Only randomness in the protocol is the Challenge → in an attack scenario if we impersonate the server,the challenge(the only randomness in the protocol) is chosen by us.
