Database Types


Any structured way of storing data can be considered a database.
• Flat-File Databases
The data is stored in a file as rows that follow an uniform pattern.
Examples: CSV files, /etc/passwd, /etc/shadow
• Relational Databases
When people referring to database, usually they are referring to Relational Database.
A Relational Database usually have different tables, each of these contains records(or rows) stored in columns.
Tables can be linked together through the use of unique keys(not passwords, just integers numbers). Primary keys are key that uniquely identifies a record in a table, this key can be used to identify that record in another table.
◇ Relational Database Management Systems (RDBMSs)
The most popular for now according to https://db-engines.com/en/ranking/relational+dbms
▪ Oracle
▪ MySQL
▪ Microsoft SQL Server
▪ PostgreSQL
▪ IBM’s Db2
▪ SQLite
▪ MariaDB
RDBMSs use their own languages called structured query language (SQL) for querying and manipulating the data contained within the databases. RDBMS to query a database use the Database Schema that is a description of the database structure written in a formal language.
• NonRelational Databases(NoSQL)
There are different categories of NoSQL or NonRelational Databases.
categories:
◇ document-oriented databases or document stores
We have to not confuse them with simple flat-file databases. document-oriented databases are more complex and flexible.
Examples: XML, YAML, JSON
▪ Document Stores Database Management Systems (NoSQL DBMSs)
The most commons according to https://db-engines.com/en/ranking/document+store
- MongoDB
- Amazon DynamoDB
- Microsoft Azure Cosmos DB
- Couchbase
◇ Key–value store
To store data associative array (also called a map or dictionary) are used as their fundamental data model
▪ Key Value Stores Database Management Systems (KV DBMSs)
The most commons according to https://db-engines.com/en/ranking/key-value+store
- Redis
- Amazon DynamoDB
- Microsoft Cosmos DB
◇ Graph DBMS
Graph DBMSs uses graph structures for semantic queries with nodes, edges, and properties to represent and store data.
▪ Graph Database Management Systems (Graph DBMSs)
The most commons according to https://db-engines.com/en/ranking/graph+dbms
- Neo4j
- Microsoft Azure Cosmos DB
