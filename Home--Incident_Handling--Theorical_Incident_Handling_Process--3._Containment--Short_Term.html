Short Term



• Try to prevent the attacker from causing more damage, while:
◇ Minimizing the changes to data on the system(s)
◇ Get untainted evidence through image creation process
• Possible short-term containment actions
◇ Disconnect network cable
◇ Pull the power cable: Loses volatile memory and may damage drive
◇ Apply filters to routers and/ or firewalls
◇ Change a name in DNS to point to a different IP address. This because most attackers target systems based on their IP address (such as 198.167.21.14) and not their domain name (www.[yourdomainname].com).
• We could use ADHD
With Active Defense Harbinger Distribution we can deploy honeypots that waste an attacker’s time, alert the administrator of the attack while still harmless and gathers information on the sources of the attack.
An useful tool included in ADHD is “Web Bug Server” for tracking when a document is opened
◇ project page: http://www.blackhillsinfosec.com/?page_id=4419
◇ download: https://github.com/adhdproject/buildkit
◇ documentation: https://adhdproject.github.io/#!index.md

ISP coordination
For external attacks, coordinate with your internet service provider
It may be able to assist you in identification, containment, and recovery. Especially for large packet floods, bot-nets, worms, and virulent spam


Creating Forensics Images
Failure to make a good working forensics image is one of the most common errors
Make forensics images of affected system(s) as soon as is practical
• Grab an image of memory as well as the file system
• Don’t do graceful shutdown, you will lose valuable data!
• Use blank media
◇ Old media often contain remnants
◇ New purchased media may have some data on it, so beware
• If possible, make a bit-by-bit image to get all file system data
• Not all incidents will allow you to do a full backup and analysis
• Tools:
◇ dd → Create binary images on UNIX and Windows machines
▪ UNIX: It is built into many UNIX and Linux distributions
▪ WINSOWS: available in many incarnations for free for Windows
◇ Rekall → Tool by Google for capturing and analyzing memory on Windows machines (https://github.com/google/rekall, http://www.rekall-forensic.com/)
◇ Volatility Framework → capture and analyze memory dumps
