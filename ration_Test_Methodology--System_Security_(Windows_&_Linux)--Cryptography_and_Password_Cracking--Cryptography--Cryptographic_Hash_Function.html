Cryptographic Hash Function


A cryptographic hash function is a deterministic algorithm that from a an input message of variable length output a block of bits of fixed length
The output is usually called as hash or digest
example of hash functions: MD5, MD4, SHA1,...

Properties of a cryptographic hash function


1. Preimage resistance: It should be infeasible starting from a given hash to find an input message
2. Second preimage resistance: it should be infeasible given an input message find another input message with the same hash
3. Collision resistance: It should be infeasible to find two different input messages with the same hash. Such a pair if found, is called a hash collision.

The properties of second preimage resistance(2) and collision resistance(3) may seem similar but the difference is that in the case of second preimage resistance,the attacker is given a message to start with, but for collision resistance no message is given; it is simply up to the attacker to find any two messages that yield the same hash value.

Avalanche effect: a single bit changed in the input message will cause a vast change in the final output hash
