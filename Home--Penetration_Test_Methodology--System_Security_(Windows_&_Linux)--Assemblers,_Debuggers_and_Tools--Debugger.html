Debugger



Useful when we have the service executable but not the source code

A debugger help us during the exploit development process, it acts as a proxy between the application and the CPU and allow us to exercise control over the application that runs in it:
• Stop the program while it is running
• Analyze the stack and its data
• Inspect registers
• Change the program or program variables and more

Most popular debuggers:
• Immunity Debugger → simple interface and allows us to use Python scripts to automate tasks
• IDA (Windows, Linux, MacOS)
• WinDBG (Windows)
• GDB (Unix, Windows)
• OllyDBG (Windows)
• X64DBG (Windows)
• Hopper (MacOS, Linux)
• EDB (Linux)
