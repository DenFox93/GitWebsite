authentication


Find subdirectories from logged user perspective


Authenticate with the web application can increase the chance of find more resources,
to test the application from the logged user perspective we can use one of these methods

• cookie option
-c <cookie_string> → Set a cookie for the HTTP request.
we can pass to dirb the cookie of the logged user
dirb https://google.com -p http://127.0.0.1:8080 -c "COOKIE:Daniele"


• Username and Password option
-u <username:password> → Username and password to use
dirb https://google.com -p http://127.0.0.1:8080 -u "admin:pass"


we can decode the Authorization header (YWRtaW46cGFzcw==) in the Burpsuite decoder panel



• Complete custom header
-H <header_string> → Add a custom header to the HTTP request
dirb https://google.com -p http://127.0.0.1:8080 -H "CustomHeader: this is my  header"
