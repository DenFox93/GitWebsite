Link files


To link the object files we ca use GoLink:
http://www.godevtool.com/GolinkFrame.htm
We can install GoLink by extracting it a put it in the same folder of NASM, because we have already put C:\Program Files\NASM in the environment variables we can use the command GoLink in every folder of our pc

example:
GoLink.exe /entry _main demo1.obj kernel32.dll user32.dll


/entry <value> → set program entry point to <value>
For example, /entry main will start the program at the code label "main:". If you do not specify the entry point with this switch GoLink assumes START will be used (in upper or lower case or a mixture). So if you want to use this default you should ensure that your code has the label "START:" at the correct place to start execution. Every EXE file needs an entry point, but in DLLs it is optional.

Kernel32.dll User32.dll → These libraries are required for the program to work correctly because the program calls functions that are defined in these libraries
These files contain the most common APIs. During the linking process GoLink looks at the list of files one by one for the required imports. When it has found all the required imports it will stop looking at the listed files.
