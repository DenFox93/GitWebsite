Penetration Test Methodology



Job of a penetration tester

:


1) Model the activities of real-world threats to discover vulnerabilities
2) Through controlled exploitation, attempt to determine the business risk associated with these flaws
3) Recommend appropriate defenses

Defining Terms
• Threat:
Someone or something that can cause harm to a target organization
Threats include:
◇ Organized crime, spyware companies, internal employees
◇ Worms and viruses also represent threats because they could cause harm in your organization
• Vulnerability: A flaw someone can exploit to cause harm
• Risk: Where threat and vulnerability overlap. A risk occurs when systems have a vulnerability that a given threat can attack.
• Exploit: Code or technique that a Threat uses to take advantage of a vulnerability


Defining terms(*) associated with what an ethical hacker or penetration tester does on a day-to-day basis


*Many people use the following terms interchangeably or without fixed meaning
• Ethical hacking:
Using computer attack techniques to find security vulnerabilities with the permission of the target owner and the goal of improving the target's security
◇ White Hat hackers are ethical hackers
◇ Black Hat hackers are sinister
◇ Grey hat has good intentions but does not have permission
• Penetration testing:
Penetration Testing is a subset of ethical hacking
While the goal of Ethical Hacking has the goal of discover vulnerabilities;
The goal of penetrating is find vulnerabilities in systems, penetrating and taking control of them
• Red Teaming:
While Penetration testers are often tasked with finding as many vulnerabilities a possible
Red Teamers are focused on:
▪ finding vulnerabilities that will help them achieve their goals
▪ Stealth and Persistence are important aspects
▪ improve the Blue Team's capabilities to detect the attack and respond to it effectively
• Vulnerability assessments (or just security assessments):
It is focused on finding vulnerabilities, which may or may not be used to get in or steal data
• Security audits:
Testing against a rigorous set of standards. The Audits are almost always done with detailed checklists


Types of Pen-Tests


• Network services test
Common types of tests, it involves finding vulnerable network services and exploit them remotely
• Assumed breach test
Find vulnerabilities in the network once an attacker has gained access to a system in the network.
Good way to find issues in.
◇ Active Directory permissions
◇ File permissions
◇ Client-side software
• Web application test
Find vulnerabilities in Web Applications
• Social engineering test
Attempt to trick a user into revealing sensitive information
• Wireless security test
Find in the target's physical environment unauthorized wireless access points or authorized wireless access points with security weaknesses
• Remote dial-up war dial test
Not common type of test today, it involves finding modems in a target environment
• Physical security test:
Find flaws in the physical security practices of a target organization.
• Stolen equipment test:
Obtain a piece of equipment, such as a laptop, from the target and then trying to extract sensitive information from it.
• Cryptanalysis attack:
This test focuses on bypassing or breaking the encryption of data stored on a local system or across the network.
• Product security test (aka shrink-wrapped software test):
Find vulnerabilities in software products
