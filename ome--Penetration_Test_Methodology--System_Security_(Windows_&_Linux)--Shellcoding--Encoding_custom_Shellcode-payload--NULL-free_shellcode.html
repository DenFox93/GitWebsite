NULL-free shellcode



If we try to use a Shellcode that use NULL characters \x00 (zeroes) against a vulnerability of type Buffer Overflow(BOF) that uses a string function (such as strcpy), the Shellcode will FAIL.
The function will stop copying the payload to the stack as soon as it encounters the first \x00 byte.

How make a shellcode NULL-free


• manual editing the shellcode so that it does not contain the string terminator.
In other words, use different instructions that perform the same operations, but do not have a string terminator
• encode and decode the shellcode
