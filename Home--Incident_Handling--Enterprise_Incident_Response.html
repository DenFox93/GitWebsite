Enterprise Incident Response


Determining the systems compromised in an enterprise with thousands of machines can seem impossible, but with the right tools and techniques.

Need to start somewhere
When we are working any incident, the first thing our responders ask for is:
• Web Proxy: the logs for the egress connections from the firewall
• DNS cache or DNS logs
◇ compare the current cache of the DNS server with a list of evil IPs and domains by using a tool like dns-blacklists.py.
A list to compare with is the one from Malware Domain List.
• Connection Logs: the logs from whatever external web-filtering device the organization is using
◇ Because malware not always makes persistent connection back to the bad guys’ command and control systems but they can also connect back at regular (or irregular) intervals.
