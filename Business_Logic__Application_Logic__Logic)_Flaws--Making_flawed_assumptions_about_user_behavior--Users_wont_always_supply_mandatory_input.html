Users won't always supply mandatory input


One misconception is that users will always supply values for mandatory input fields.
The presence or absence of a particular parameter may determine which code is executed. Removing parameter values may allow an attacker to access code paths that are supposed to be out of reach


When probing for logic flaws we need to:
• remove one parameter at a time and ensure all relevant code paths are reached.
• Try deleting entirely a parameter or only its value. The server usually handle the two cases differently.
• Sometimes tampering with a parameter in one step will have an effect on another step further along in the workflow.
• We can apply this process to:
◇ URL parameters
◇ POST parameters
◇ cookies parameters
