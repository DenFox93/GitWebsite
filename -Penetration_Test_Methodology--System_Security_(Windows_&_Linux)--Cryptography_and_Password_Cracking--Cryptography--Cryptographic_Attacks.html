Cryptographic Attacks


The goal of Cryptographic Attacks is exploit weaknesses of crypto algorithms to decipher ciphertext without knowing the key

Classification of Cryptographic Attacks:
• Known Only Attack
◇ Known Plaintext only attack: access to the plaintext and the corresponding ciphertext
◇ Known Ciphertext only attack: known only the ciphertext (no plaintext)
• Chosen Attack
◇ Chosen Plaintext attack: the attacker has the ability to choose plaintexts and to view their corresponding encryption ciphertexts.
The limitation is that is possible to do only a request(with inside more plaintexts)
◇ Chosen Ciphertext attack: attacker has the ability to choose ciphertexts and to view their corresponding decryption plaintexts.
The limitation is that is possible to do only a request(with inside more chipertexts)
It is essentially the same scenario as a Chosen Plaintext Attack but applied to a decryption function, instead of the encryption function
• Adaptive Chosen Attack
◇ Adaptive chosen plaintext attack: it is a Chosen Plaintext Attack scenario in which the attacker has the ability to make his choice of the inputs to the encryption function
Differently from Chosen Plaintext attack is possible to do more request and therefore is possible base the new requests on previous responses
◇ Adaptive chosen ciphertext attack: it is a Chosen Ciphertext Attack scenario in which the attacker has the ability to make his or her choice of the inputs to the decryption function
Differently from Chosen Chiphertext attack is possible to do more request and therefore is possible base the new requests on previous responses
