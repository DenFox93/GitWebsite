Create custom dictionaries


1. We should augment our dictionary with specialized words used by the target organization, including its product and service names, people's names associated with the organization, and so on. In effect, we should create a customized dictionary specially crafted for each project.
To do that we can use a software to crawl the target's websites, creating a local mirror of its pages.
◇ To do that we can use CeWL
2. Take this CeWL word list and prepend it to your normal word list to create a more fitted dictionary.
cat CeWLwordlist.txt anotherDictionary.txt >finalWordlist.txt

3. Make use of mangling rules, that modify little bit the words(example: password could be mangled into Password, password1, Password1, p@ssw0rd, or P@ssw0rd1)
◇ To do that use Rsmangler
4. Then if you can get a copy of its password policy, customize your word list to match its policy (minimum number of characters, special character requirement, and such).
◇ To do that we can use pw-inspector
5. Make sure it contains only unique words. Any duplicated words only waste valuable time
cat wordlist.txt | sort | uniq > dictionary.txt
